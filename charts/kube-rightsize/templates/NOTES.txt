kube-rightsize has been installed.

1. Verify the operator is running:

   kubectl get pods -n {{ .Release.Namespace }} -l {{ include "kube-rightsize.selectorLabels" . | replace ": " "=" | replace "\n" "," }}

2. Check operator logs:

   kubectl logs -n {{ .Release.Namespace }} deploy/{{ include "kube-rightsize.fullname" . }} -f

3. Create your first RightSizePolicy:

   cat <<EOF | kubectl apply -f -
   apiVersion: rightsize.io/v1alpha1
   kind: RightSizePolicy
   metadata:
     name: my-policy
     namespace: default
   spec:
     targetRef:
       kind: Deployment
       name: my-app
     metricsSource:
       prometheus:
         address: http://prometheus-server.monitoring:80
       historyWindow: 168h
     cpu:
       percentile: 95
       overhead: "20"
       minAllowed: 50m
       maxAllowed: "4000m"
     memory:
       percentile: 99
       overhead: "30"
       minAllowed: 64Mi
       maxAllowed: 8Gi
     updateStrategy:
       type: Recommend
   EOF

4. Check policy status:

   kubectl get rightsizepolicies -A

{{- if .Values.initialSizing.enabled }}
5. Initial sizing is enabled. To activate it for a namespace:

   kubectl label namespace <your-namespace> rightsize.io/initial-sizing=enabled

   Then set initialSizing: true on your RightSizePolicy's updateStrategy.
   New pods will get recommended resources at creation time.
{{- end }}
